Description:
Abstract:
This dataset contains two sources used to determine X, Y coordinates for positions along Washington State highways perpendicular to milepost markers. Locations were collected on the fly from a moving vehicle using Global Positioning Systems (GPS) technology, and using Geographic Information Systems (GIS) processes, the Washington State Department of Transportation (WSDOT) 24k Linear Referencing System (LRS) and the WSDOT Traffic Signs Management Systems database (TSMS).
Purpose:
Provide location reference along Washington State Highways for such uses as: routing, vehicle tracking, emergency services, traffic safety, and WSDOT project planning and management.

Completeness_Report:
This dataset represents all State Routes in Washington State. However, milepost markers are not present for every mile. Milepost markers were collected in the field or from a query of the TSMS database.
Positional_Accuracy:
Horizontal_Positional_Accuracy:
Horizontal_Positional_Accuracy_Report:
This dataset contains two sources used to determine X,Y coordinates for positions along state highways perpendicular to milepost markers. The dataset contains a field indicating the source. Source 1 was collected with GPS from a moving vehicle as a single point. Source 2 was obtained through GIS processes using a 24k LRS and data from the Traffic Sign Management System. Source 1 data points with a directional indicator of 'b' have an average accuracy of +/- one-hundredth of a mile (52.8 ft.). Source 1 data points with a directional indicator of 'i' or 'd' have an average accuracy of +/- two-hundredths of a mile (105.6 ft). Source 2 data points have an unknown accuracy.

Process_Step:
Process_Description:
Initial data collection was performed with GPS technology on the fly from moving vehicles. Data was collected in the increasing and decreasing direction of travel as a single point while passing perpendicularly to each milepost marker. The milepost markers along undivided highways were collected twice, once in each direction where possible. The milepost markers along divided highways were collected once in the direction of travel, where possible.
The GPS data was post processed and exported with Pathfinder Office into a text file for each state route. New data fields were added: State Route ID (identification) and ARM (actual route mileage) values were derived from the WSDOT 24k LRS dataset. The increasing and decreasing files of each route were joined together geospatially using the nearest function, and two tables were produced. This data was then validated, the average distance field was created, and a single table was produced. An SRMP value was obtained using the ArmCalc module to determine the MP_value.
For source 2 mileposts, the TSMS database was queried for milepost markers. This dataset was compared to the source 1 data and duplicates were removed. ARM values were obtained from the SRMP values given in the TSMS database using ARMCalc and then placed on the WSDOT's 24k LRS. X,Y coordinates were obtained for these points using ESRI's GIS software.
The source 1 and source 2 data sets were combined into a single file, placed as an event theme using the longitude and latitude coordinate fields, and then exported as a shapefile. This shapefile was projected from GCS NAD83 to GCS NAD 83 HARN. New NAD 83 HARN coordinates were generated with the ESRI AddXY command and loaded into the longitude and latitude fields.
